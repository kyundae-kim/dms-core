# 메시지 정의서

## 현재 범위

현재 SDK는 메시지 브로커 연계 API나 도메인 이벤트 발행 계약을 제공하지 않습니다. 문서 등록·조회·삭제·복구 결과는 동기식 반환값과 공개 예외로 전달됩니다.

## 운영 관찰

SDK는 주요 처리 결과를 구조화된 로그로 기록합니다. 복구 작업은 `recovery_audit_hook`을 통해 애플리케이션의 감사 로그 또는 메시징 계층에 연결할 수 있습니다. 이 hook은 best-effort이며, hook 실패가 원래 복구 결과를 변경하지 않습니다.

메시지 브로커 발행, 전달 보장, 재처리, 순서 보장 및 메시지 스키마 버전 관리는 호스트 애플리케이션의 책임입니다. 향후 SDK가 메시징 계약을 제공하게 되면 본 문서에 이벤트 이름, payload, 전달 의미와 호환성 정책을 정의해야 합니다.
